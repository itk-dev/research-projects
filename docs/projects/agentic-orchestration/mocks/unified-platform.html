What's the difference between creating a process and an automation?
Good question. A process is a multi-step flow that may involve humans, AI agents, and integrations — think of it as the full end-to-end workflow (e.g., handling a complaint from submission to resolution). It's modeled in BPMN and managed by the process engine.

An automation is a technical integration between systems — it moves data, triggers actions, or connects APIs. Automations are the building blocks that processes use. For example, "fetch case from SBSYS" is an automation; "handle parking complaint" is a process that uses that automation.

If you're unsure which to create, click + New Process or + New Automation and choose "Chat about it" — I'll help you figure out the right approach.
